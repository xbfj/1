fire.css

fire.css is a fire-themed and beautiful CSS stylesheet. Its made to make your websites look more responsive, vibrant and cooler. You can read more at About, read our FAQ if you have any questions, or look at our demo for an example of our stylesheet.

Get started

Create a new index.html file in your project root, and copy the HTML code below you'll need to get started.

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Title</title>
    <link href="https://firecss.hecker40.repl.co/fire.css" rel="stylesheet" type="text/css"/>
    <link href="https://website.com/site.css" rel="stylesheet" type="text/css"/>
  </head>
  <body>
    <main>
    </main>
    <script src="https://website.com/site.js"></script>
  </body>
</html>

This will be the basic structure of our HTML until we start adding more code. Note that https://website.com/site.css and https://website.com/site.js are just examples, in your actual project you would remove them and replace the URLs to your own scripts and stylesheets.

Be sure to place your navbar in front of your <main> tag and not inside it, or else the navbar won't work.

Buttons

A default and secondary button.

<button>Default button</button>
<button class="btn-secondary">Secondary button</button>

Our toolkit also includes buttons of all sizes, including big and small ones. As a side note, you can combine classes (such as btn-secondary and btn-small to make a secondary and small button, as demonstrated below.

<button class="btn-large">Large button</button>
<button class="btn-secondary btn-small">Small button</button>

Make buttons look inactive by adding the disabled state to any <button> element.

<button disabled>Disabled button</button>
<button class="bg-secondary" disabled>Disabled button</button>

Have too much text that won't fit the button? Don't worry, we also have block buttons, or long buttons that take up a lot 100% of the device width to ensure that your text won't overfill.

<button class="btn-block">Block button</button>

Cards

Cards, built with flexbox! The flexbox will fit two cards per row, but you can change that by adding flex: 0 1 <percentage>, where the percentage is one divided by the number of cards you want per row minus 2-3% for the gap space of each card. Note that any cards extra will be displayed as the full, body width.

  <div class="card">

FAQ

I've made a FAQ part to this CSS stylesheet in case you had any questions that you were wondering about. I'll answer the most common ones here.

Is this CSS stylesheet free?

Yes, fire.css is an open-source project and is completely free.

Do I need to credit the author if I use this?

Yes, it is preferred that you give credit to me if you use it, either in the website itself or in the repl description. Reading this CSS stylesheet's MIT License will probably clarify most of your questions.

Can I change the color scheme?

Yes, you can. Fork the repl and open fire.css in your editor. You can then see the main variables in :root and change them, this will directly change the colors.

You can also open Inspect (Ctrl Shift I) and change the CSS from there. From there, you can directly edit the CSS colors and make it your own. An example is shown below.

Can I see a demo of fire.css?

Yes, you can click on this link for a demo of a website using fire.css.

What's special about fire.css?

Most CSS stylesheets don't have a theme; all their components are plain and aren't considered "modern". This stylesheet uses a lively color scheme to make your website more catchy to users when they view it.

How can I help?

You can help by going to the repl's cover page and leaving some feedback, all feedback is appreciated. You can also report a bug or anything else you think would make this a better stylesheet for websites to use. If you want, you can also help by sharing this to your friends and liking the repl, as it will definitely help more people if you do that.

Hope you all found this useful!
